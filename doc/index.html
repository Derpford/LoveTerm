Module loveterm

LoveTerm - simple 1-bit tiled graphics for Love2d

Usage:

    local loveterm = require "loveterm"
local codes = require "extra/cp437"
local color = require "extra/color"

function love.load()
  screen = loveterm.create("tilesets/CGA8x8thick.png", 80, 40)
  screen:print(
      screen.wrapString(
          "LoveTerm is a small library for drawing 1-bit " ..
          "tiled graphics, such as terminal emulators.",
          21),
      5, 5)
  screen:set(codes.heart, color.c64.pink, screen.defaultbg, 3, 5)
end

function love.draw()
  screen:draw()
end

rectangle (mode, x, y, w, h, options) line 262
Adds a rectangle to the screen.

The options table has the fields fg and bg to set the colors, as well as topLeft, topRight, bottomLeft, bottomRight, vertical, and horizontal to set the various components of the rectangle when mode is set to line. options also has a fill field to set which character is drawn when mode is fill.

Parameters:

  • mode can either be "fill" or "line"
  • x int
  • y int
  • w int the width of the rectangle in cells
  • h int the height of the rectangle in cells
  • options table

Draw an image to the screen.

The options table can include setfg (default: false), setbg (default: true), filter (default: "linear", other possible setting is "nearest"), or alpha (default: 255), which is the alpha to draw the image with.

  • options table the options table (default {})
  • x int the x coordinate of the screen to start drawing on (default 0)
  • y int the y coordinate of the screen to start drawing on (default 0)
  • r int orientation in radians (default 0)
  • sx int scale-factor (x-axis) (default 1)
  • sy int scale-factor (y-axis) (default sx)
  • ox int origin offset (x-axis) (default 0)
  • oy int origin offset (y-axis) (default 0)
  • kx int shearing factor (x-axis) (default 0)
  • ky int shearing factor (y-axis) (default 0)
